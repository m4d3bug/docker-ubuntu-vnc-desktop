docker-ubuntu-vnc-desktop
=========================

Docker image to provide HTML5 VNC interface to access Ubuntu 24.04 LXDE desktop environment, with built-in Chromium, Firefox, Node.js 22, and chrome-devtools-mcp support.

Forked from [fcwu/docker-ubuntu-vnc-desktop](https://github.com/fcwu/docker-ubuntu-vnc-desktop) and upgraded.

Quick Start
-------------------------

Run the docker container and access with port `6080`

```
docker run -p 6080:80 -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

Browse http://127.0.0.1:6080/

VNC Viewer
------------------

Forward VNC service port 5900 to host by

```
docker run -p 6080:80 -p 5900:5900 -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

Now, open the vnc viewer and connect to port 5900. If you would like to protect vnc service by password, set environment variable `VNC_PASSWORD`, for example

```
docker run -p 6080:80 -p 5900:5900 -e VNC_PASSWORD=mypassword -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

A prompt will ask password either in the browser or vnc viewer.

Chrome DevTools MCP
------------------

The image includes `chrome-devtools-mcp` running as a service, with a Chromium instance accessible via Chrome DevTools Protocol (CDP).

- CDP is available inside the container at `localhost:9222`
- Exposed externally via socat proxy on port `9223`

```
docker run -p 6080:80 -p 9223:9223 -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

Verify CDP access:

```
curl http://127.0.0.1:9223/json/version
```

HTTP Base Authentication
---------------------------

This image provides base access authentication of HTTP via `HTTP_PASSWORD`

```
docker run -p 6080:80 -e HTTP_PASSWORD=mypassword -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

SSL
--------------------

To connect with SSL, generate self signed SSL certificate first if you don't have it

```
mkdir -p ssl
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout ssl/nginx.key -out ssl/nginx.crt
```

Specify SSL port by `SSL_PORT`, certificate path to `/etc/nginx/ssl`, and forward it to 6081

```
docker run -p 6081:443 -e SSL_PORT=443 -v ${PWD}/ssl:/etc/nginx/ssl -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

Screen Resolution
------------------

The Resolution of virtual desktop adapts browser window size when first connecting the server. You may choose a fixed resolution by passing `RESOLUTION` environment variable, for example

```
docker run -p 6080:80 -e RESOLUTION=1920x1080 -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

Default Desktop User
--------------------

The default user is `root`. You may change the user and password respectively by `USER` and `PASSWORD` environment variable, for example,

```
docker run -p 6080:80 -e USER=doro -e PASSWORD=password -v /dev/shm:/dev/shm m4d3bug/ubuntu-desktop-lxde-vnc
```

Deploy to a subdirectory (relative url root)
--------------------------------------------

You may deploy this application to a subdirectory, for example `/some-prefix/`. You then can access application by `http://127.0.0.1:6080/some-prefix/`. This can be specified using the `RELATIVE_URL_ROOT` configuration option like this

```
docker run -p 6080:80 -e RELATIVE_URL_ROOT=some-prefix m4d3bug/ubuntu-desktop-lxde-vnc
```

NOTE: this variable should not have any leading and trailing splash (/)

Generate Dockerfile from jinja template
-------------------

Dockerfile and configuration can be generated by template.

- arch: one of `amd64`, `arm64` or `armhf`
- flavor: refer to file in flavor/`flavor`.yml
- image: base image (default: `ubuntu:24.04`)
- desktop: desktop environment which is set in flavor
- addon_package: Debian package to be installed which is set in flavor

```
make clean && IMAGE=ubuntu:24.04 FLAVOR=lxde make build
```
